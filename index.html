Neural Networks are Not Scary
✓ Completed
Core Idea

A neural network adjusts weights and biases to make the model fit the data better. Training is the process of finding the right values for these two things.

Weight vs Bias

Weight — how much each input matters (the slope)

Bias — shifts the activation threshold (the intercept)

  • Neuron — multiplies inputs by weights, adds bias, then passes through an activation function
  • Activation function — introduces non-linearity so the network can learn complex, curved patterns — not just straight lines
  • Training goal — find the weights and biases that minimize prediction error on the training data
  • Why "not scary" — even the most complex networks are just many of these simple neurons stacked together
Interview Q&A
Explain a neural network in simple terms.
A neural network learns by adjusting weights and biases. Weights control how much each input contributes to the output, and biases shift the activation threshold. Training is the process of finding the weight and bias values that minimize prediction error across the training data.
What is the role of an activation function?
Without an activation function, stacking multiple layers is mathematically equivalent to a single linear layer — you can only model linear relationships. Activation functions add non-linearity, allowing the network to approximate arbitrarily complex functions.
🔬 In my LSTM model for neonatal pain detection, the weights learned how much each physiological signal — heart rate, facial action units — contributes to the pain prediction at each timestep. The bias terms help calibrate the activation threshold for each hidden neuron.

Prerequisite · StatQuest with Josh Starmer
The Chain Rule
✓ Completed
Core Idea

When variables are not directly connected, the chain rule lets us multiply the derivatives of each intermediate step to find the total effect — chaining them from start to finish.

The Formula

If y = f(x) and x = g(z), then:

dy/dz = dy/dx · dx/dz

Multiply the derivatives of each step together.

  • Why we need it — in a neural network, an input weight is not directly connected to the final loss. There are many layers in between. The chain rule lets us trace the effect through every intermediate layer.
  • How it works — break the path into steps, compute the derivative of each step, then multiply them all together.
  • Why it matters for backprop — backpropagation is just the chain rule applied repeatedly, from the output loss all the way back to the first layer's weights.
Interview Q&A
What is the chain rule and why does backpropagation need it?
Neural networks have many layers between the input and the output. When we want to know how a change in one weight affects the final loss, they're not directly connected — there are many intermediate layers in between. The chain rule lets us multiply the derivatives of each step together, chaining them from the output all the way back to that weight. Without it, we couldn't compute gradients through deep networks.
🔬 In my LSTM model for neonatal pain detection, backpropagation through time (BPTT) applies the chain rule across every timestep — from the final pain prediction loss, backwards through each hidden state, all the way to the weights that processed the first frame of the video.
